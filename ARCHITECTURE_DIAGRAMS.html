Key Components:
  • Client Layer: Web browsers accessing 3 role-specific dashboards
  • Frontend: Next.js 14 with React 18, Tailwind styling, Recharts analytics
  • Authentication: NextAuth.js handling JWT sessions and credentials
  • Backend: Serverless API routes for goals, approvals, check-ins, admin, reports

7-Phase Workflow:
  • Phase 1: Employee creates goal (draft saved)
  • Phase 2: Employee submits (validates 100% weightage, notifies manager)
  • Phase 3: Manager approves/rejects (locks or returns for rework)
  • Phase 4: Employee logs quarterly progress (score auto-calculated)
  • Phase 5: Manager adds feedback comments
  • Phase 6: Analytics aggregated and reports exported
  • Phase 7: All changes audited with before/after snapshots

5. Database Schema Relationships

Core entities and their relationships in the PostgreSQL database managed by Prisma ORM. 
User (Employee/Manager/Admin)
├── Department (organizational unit)
├── Manager (reporting hierarchy)
├── GoalSheet (1 per user per cycle)
│   ├── Goals (1-8 per sheet)
│   │   ├── QuarterlyCheckIn (Q1-Q4 progress)
│   │   └── SharedGoal reference
│   └── Approver (manager who approved)
├── SharedGoals (departmental KPIs)
│   └── Goals linked across employees
├── AuditLogs (post-lock changes)
├── Notifications (in-app alerts)
└── Escalations (workflow escalations)

Cycle (annual performance year)
├── CheckInWindows (GOAL_SETTING, Q1-Q4)
└── GoalSheets & SharedGoals (linked to cycle)

Relationships:
• User → Department (many:1)
• User → Manager (many:1, self-referential)
• GoalSheet → Goal (1:many)
• Goal → QuarterlyCheckIn (1:many)
• Goal → SharedGoal (many:1)
• Cycle → CheckInWindow (1:many)

6. Deployment Architecture

Development Environment:
  • Node.js 18+ on local machine
  • PostgreSQL running in Docker (docker compose up -d)
  • Next.js dev server (npm run dev) on http://localhost:3000
  • Hot reload on file changes
  • Database migrations via Prisma
Production Environment:
  • Frontend & Backend: Vercel (serverless functions, auto-scaling, global CDN)
  • Database: Neon PostgreSQL (managed cloud, connection pooling)
  • CI/CD: GitHub push → Vercel build → Deploy
  • Environment: Configured in Vercel dashboard (DATABASE_URL, NEXTAUTH_SECRET)
  • Regions: 40+ worldwide for low latency
